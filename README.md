# 华夏神话 · Huaxia Myth — 中国神话可交互知识引擎

A bilingual, SSR-first, interactive knowledge engine for Chinese mythology:
a Shan Hai Jing (《山海经》) bestiary, a force-directed genealogy graph, and
open structured data. Product brief: [doc/myth-brd.md](doc/myth-brd.md).

## Stack

- [**React Router v7**](https://reactrouter.com/) (framework mode) — SSR + nested
  routes + `loader`/`meta`. Server-rendered HTML carries titles, hreflang and
  schema.org JSON-LD for SEO (the BRD's #1 growth lever).
- [**React 19**](https://react.dev/) + [**Vite**](https://vite.dev/)
- [**Hono**](https://hono.dev/) — the data API, mounted at `/api/*`.
- [**Cloudflare Workers**](https://developers.cloudflare.com/workers/) via the
  Cloudflare Vite plugin (streaming SSR with `renderToReadableStream`).
- [**d3-force** style](https://github.com/d3/d3-force) self-contained canvas
  force simulation for the genealogy graph.

## Project layout

```
app/
  root.tsx              # <html> shell + LangProvider + Nav
  entry.server.tsx      # streaming SSR (renderToReadableStream)
  routes.ts             # route table
  routes/               # home · bestiary · creature.$id · graph · atlas · quiz
                        # about · og/creature.$id · sitemap · robots
  components/           # Nav LangToggle Footer SealStamp CreatureCard FilterChips
                        # ShareCard ForceGraph ConstellationCanvas AtlasMap
  data/                 # types · dict · creatures · graph · atlas · repo (★ seam)
  i18n/LangContext.tsx  # zh|en, cookie-driven SSR + localStorage persistence
  lib/                  # seo (meta/hreflang/JSON-LD) · drawCard · apiGuard
  styles/               # tokens.css · app.css · pages.css
workers/app.ts          # Hono (/api over D1) + React Router request handler
migrations/             # D1 schema + seed (generated by scripts/gen-seed.ts)
scripts/gen-seed.ts     # emits migrations/0001_init.sql from the typed TS data
```

`app/data/repo.ts` is the data-access seam: every function takes an optional D1
binding — with it, queries Cloudflare D1; without it, falls back to the local
typed TS data. Route loaders pass `context.cloudflare.env.DB`; the Hono `/api`
uses the same functions. Source flips with no change to components.

## Data API

Read-only JSON over Hono (metered by `app/lib/apiGuard.ts`):

```
GET /api/creatures        all creatures
GET /api/creatures/:id    one creature
GET /api/graph            genealogy nodes + links
GET /api/health           liveness
```

API-key auth is enforced only when the `API_KEYS` var is a non-empty,
comma-separated allowlist (clients send `x-api-key`); rate limiting (120
req/min) is enforced only when the `RATE_LIMIT` KV namespace is bound.

## Development

```bash
pnpm install
pnpm dlx wrangler d1 migrations apply DB --local   # seed local D1 (once)
pnpm dev                                            # http://localhost:5173
```

Regenerate the seed after editing `app/data/creatures.ts` / `graph.ts`:

```bash
node scripts/gen-seed.ts                            # rewrites migrations/0001_init.sql
pnpm dlx wrangler d1 migrations apply DB --local    # re-apply
```

## Content & licensing (hard lines, BRD §7)

- Original text only from the **public-domain《山海经》**.
- **Do not** introduce Yuan Ke's (袁珂) retellings, classifications or dictionary
  entries; **do not** scrape ctext.org value-added/RDF data for commercial use.
- Ancient illustrations are placeholders ("ART SLOT"); replace with verified
  Wikimedia Commons public-domain scans.

## Roadmap

**Phase 1 (done)** — SSR site, 4 pages, bilingual i18n, SEO baseline.

**Phase 2 (done)** — D1 + migrations seeded from TS; Hono `/api` over D1 with the
`repo.ts` seam; SEO hardening (breadcrumb + ItemList JSON-LD, related creatures,
per-creature OG image at `/og/creature/:id`).

**Phase 3 (done)** — MapLibre interactive atlas (`/atlas`); "which creature are
you" quiz (`/quiz`); public data API with KV rate-limit + optional API-key auth;
data & licensing page (`/about`).

**Deferred — needs external resources / business setup (not code):**
- Content scale to 50–100 entries via a Wikidata (CC0) import + human review.
- OG image as **PNG** (Satori + resvg-wasm with a bundled Noto SC subset) for full
  Twitter/Facebook compatibility — current `/og/creature/:id` returns SVG.
- Monetization: display ads (Mediavine/Raptive), merch (poster/print), and
  membership (爱发电/Patreon) — all require external accounts and traffic
  thresholds. See [doc/myth-brd.md](doc/myth-brd.md) §6.

To enable API metering in production: bind a real `RATE_LIMIT` KV namespace and
set `API_KEYS` to a comma-separated allowlist in `wrangler.json`.
